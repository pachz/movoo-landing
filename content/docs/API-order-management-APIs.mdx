# Endpoints (Shop → Muvu)

## 1. Create Order  
`POST /v1/orders`

Creates a new delivery order in Muvu.

### Request Body

```bash
{
  "integration_order_id": "SHOP123456",
  "pickup": {
    "name": "Store Name",
    "address": "789 Elm St, Kuwait City",
    "lat": 29.3375,
    "lng": 48.0286,
    "phone": "+96500000000"
  },
  "dropoff": {
    "name": "Customer Name",
    "address": {
      "area": "Salmiya",
      "block": "3",
      "street": "Al Muqawil",
      "building": "21A",
      "floor": "1",
      "flat": "2"
    },
    "lat": 29.3759,
    "lng": 47.9774,
    "phone": "+96511111111"
  },
  "items": [
    {
      "name": "Product A",
      "quantity": 2,
      "price": 1.25,
      "weight": 0.53
    }
  ],
  "notes": "Leave at the door",
  "cod_amount": 3.5,
  "auto_pickup": true,
  "scheduled_pickup_time": "2025-07-16T15:00:00Z"
}
```

> **Note**: address fields can be a plain string or a structured object. lat and lng are optional but improve delivery accuracy.

#### 200 OK Response
```bash
{
  "success": true,
  "id": "MOV123",
  "status": "created",
  "tracking_url": "https://sample.com/track/MOV123",      // optional
  "status_url": "https://api.sample.com/v1/orders/MOV123/status"
}
```

#### 400 Bad Request
```bash
{
  "success": false,
  "error": "validation_error",
  "message": "Missing required field: dropoff.phone"
}
```

#### 409 Conflict
```bash
{
  "success": false,
  "error": "duplicate_order",
  "message": "An order with this integration_order_id already exists."
}
```

## 2. Cancel Order  
`POST /v1/orders/{id}/cancel`

Cancels a previously created order.

#### 200 OK Response
```bash
{
  "success": true,
  "id": "MOV123",
  "status": "cancelled"
}
```

#### 404 Not Found
```bash
{
  "success": false,
  "error": "not_found",
  "message": "Order not found"
}
```

#### 400 Bad Request
```bash
{
  "success": false,
  "error": "invalid_action",
  "message": "Cannot cancel an order that has already been delivered."
}
```

---

## 3. Get Order Status  
`GET /v1/orders/{id}/status`

Fetches the latest delivery status of a given order.

#### 200 OK Response
```bash
{
  "success": true,
  "id": "MOV123",
  "integration_order_id": "SHOP123456",
  "status": "in_transit",
  "last_updated": "2025-07-16T09:42:00Z"
}
```

#### 404 Not Found
```bash
{
  "success": false,
  "error": "not_found",
  "message": "Order not found"
}
```

---

## 4. Trigger Pickup  
`POST /v1/orders/{id}/pickup`

Manually triggers a pickup for an existing order if auto-pickup is disabled or delayed.

#### 200 OK Response
```bash
{
  "success": true,
  "id": "MOV123",
  "status": "dispatched"
}
```

#### 404 Not Found
```bash
{
  "success": false,
  "error": "not_found",
  "message": "Order not found"
}
```

#### 400 Bad Request
```bash
{
  "success": false,
  "error": "invalid_status",
  "message": "Pickup cannot be triggered. Current status: picked_up"
}
```
