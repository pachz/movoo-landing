## Setup Instructions

You have **two options** to configure the environment:  

#### Option A ‚Äî Import the Environment (Recommended)  
- Open **Postman** ‚Üí **Environments** ‚Üí **Import**  
- Upload the file `Muvx Env.postman_environment.json`  
- Activate the environment named **Muvx Env** from the dropdown in the top-right  

Environment variables included:  

- `muvx_base_url` ‚Üí API base URL (`https://core.muvx.ai`)  
- `sample_api_key` ‚Üí **Replace this with your own API key**  
- `sample_order_id` ‚Üí **First create an order using the API, then use its ID here**  

<Banner>
- Always replace the <code>sample_api_key</code> value with your actual API key.<br/>
- To set <code>sample_order_id</code>, first use the <b>Create Order</b> API, then copy the returned <code>id</code> into this variable.
</Banner>

- Open **Postman** ‚Üí **Collections** ‚Üí **Import**.

## Authentication

This collection uses **Bearer Token authentication**:  
- The token is set at the collection level.  
- All requests automatically use `{{sample_api_key}}` from your active environment.  
- You don‚Äôt need to edit headers manually.  

---

## Using the Collection

With the environment and collection active, you can immediately start testing:

- **Create Order** ‚Üí Register a new order in Muvx  
- **Trigger Pickup** ‚Üí Dispatch an order to delivery  
- **Get Order Status** ‚Üí Track status updates  
- **Cancel Order** ‚Üí Cancel an existing order
